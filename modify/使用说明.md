# 透明代理架构：文件存储与目录说明

在使用全新的透明代理架构时，服务器端、客户端以及本地代理端的文件存储职责和目录划分如下：

## 1. 远端云服务器 (Linux Server)
所有耗费显存的**模型文件**、**从本地上传的临时素材** 以及 **生成的计算结果视频** 都储存在这里。
*   **根目录定位**：根据我们编写的 `start_cloud_server.sh` 脚本，远端数据根目录被默认指定为了项目根目录下的 `ltx_cloud_data/`。
*   **子目录职责明细**：
    *   `ltx_cloud_data/models/`：大模型文件（你的 Linux 在第一次跑模型时下载的几十 GB 权重都在这，请确保云端有足够的硬盘空间）。
    *   `ltx_cloud_data/uploads/`：上传素材缓存（你的本地代理拦截并上传的参考图片、首尾帧视频等存放在这，生成完毕后你可以定期写脚本清理该目录释放空间）。
    *   `ltx_cloud_data/outputs/`：生成结果缓存（云端生成的各种 `.mp4` 视频存放处，供本地代理下载拉取）。

## 2. 本地透明代理端 (Windows Proxy)
代理端主要负责做文件“搬运工”。它不下载模型，但需要把云端生成的视频**拉取**回本地，方便前端组件（如 Timeline 剪辑器）实现秒级无缝回放。
*   **根目录定位**：代理端由 Electron 里的 `python-backend.ts` 唤醒启动，并且它会拿到官方原生地应用数据目录变量（`LTX_APP_DATA_DIR`）。通常在你 Windows 电脑的：
    `C:\Users\你的用户名\AppData\Roaming\LTX-Desktop\`
*   **子目录职责明细**：
    *   `AppData\Roaming\LTX-Desktop\outputs\`：回传下载的视频与渲染结果统统保存在这里。如果以后你想提取以前生成的某段视频原文件，可以直接去这个文件夹找。
    *   *(注：由于代理通过拦截了 `status` 探针伪装成了“模型健康状态”，所以代理端所在的电脑不会产生 `/models` 的庞大占用)*

## 3. 本地客户端 UI (Windows Frontend)
前端也就是你看到的 React 图形界面，**它本身不产生专属的本地媒体存储目录**：
*   前端导入用到的所有素材，都是你从本机关联的具体绝对物理路径（比如你从桌面上拖进去的 `C:\Users\Desktop\测试视频.mp4`）。
*   生成播放的视频，也会直接按照 `file://` 协议，去读取上面第二点中提到的代理端存在 `outputs\` 里的落地成果。
*   配置参数等配置文件 (`settings.json`) 会保持保存在 `AppData\Roaming\LTX-Desktop\` 根目录中。

---

> **一句话总结**：
> 大模型和上传的临时文件等脏活累活都在 Linux 的 `ltx_cloud_data` 目录里承担；而你最终期待的成片视频，会被悄悄拖回 Windows 电脑的 `AppData\Roaming\LTX-Desktop\outputs` 目录下妥善永存。前端直接读取本地视频，体验极致丝滑且电脑干净！
